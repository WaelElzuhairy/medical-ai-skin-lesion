The Model — ViT-Base-16

How ViT Actually Works

Input image (224×224)
    │
    ▼
Split into 14×14 grid → 196 patches of 16×16 pixels each
    │
    ▼
Each patch → linear embedding → 768-dimensional vector
    │
    ▼
Add [CLS] token (learned) + positional embeddings
    │
    ▼
12 × Transformer Encoder Layers
    Each layer: Multi-Head Self-Attention + MLP + LayerNorm
    # every patch attends to every other patch — global context
    │
    ▼
[CLS] token output (768-dim) → 7-class linear head
    │
    ▼
Logits → Temperature Scaling → Softmax → Probabilities

Why ViT Instead of a Standard CNN?

A CNN's receptive field grows gradually through layers — it can't model long-range relationships until deep in the network. ViT uses self-attention: every patch attends to every other patch from layer 1. For dermoscopy, the relationship between border texture and center pigmentation is diagnostically important. Self-attention captures that directly.

Calibration — The Reliability Argument

💡 This is where most teams get exposed. The router uses confidence scores to make routing decisions. If those scores are wrong, the whole pipeline is wrong. Temperature scaling is what makes them trustworthy.

What is ECE (Expected Calibration Error)?

A model that says "90% confidence" should be right exactly 90% of the time. ECE measures how far off it is.

# How it's computed:
# 1. Bin all predictions into 10 buckets: 0-10%, 10-20%, ..., 90-100%
# 2. For each bucket: measure avg confidence vs actual accuracy
# 3. ECE = weighted average of |avg_confidence - accuracy| per bucket

ECE = 0.0  # perfect calibration (impossible in practice)
ECE = 0.030  # our vit_v4 — confidence is off by only 3% on average ✅
ECE = 0.15  # typical uncalibrated neural network

What is Temperature Scaling?

# Raw neural networks are systematically overconfident
# They say 95% even when they're wrong

# Fix: divide logits by scalar T before softmax
calibrated_probs = softmax(logits / T)

# T > 1 → softer distribution (less confident)
# T < 1 → sharper distribution (more confident)
# T = 1 → no change (raw softmax)

# Our vit_v4: T = 0.583
# Meaning: the model was slightly underconfident, we sharpened it slightly
# Fitted on validation set using LBFGS to minimize NLL loss
# The test set never sees T during fitting — no data leakage

Why Does This Matter for the Router?

The confidence router makes a clinical routing decision based on the confidence score. If the raw softmax says "92% confident" but the model is actually right only 60% of the time at that level, your HIGH tier cases are actually unreliable. Temperature scaling ensures that when the router sees confidence > 0.85, that number actually means what it says.

The 8-Agent Pipeline

Why an Agentic Architecture?

Each agent has a single, auditable responsibility. If something goes wrong in a report, you can trace exactly which agent produced the faulty output. A monolithic "ask GPT-4 to give a diagnosis" approach is a black box — you can't audit, test, or constrain it.

🔀 Confidence Router
Pure Python rule: <50% → LOW, 50-85% → MEDIUM, >85% → HIGH. No LLM. Unit-tested at every boundary.
RULE-BASED ONLY
🔬 Diagnosis Agent
Takes CNN output + patient metadata. LLM cross-references them. Returns strict JSON: agrees, contradictions, rationale. Schema validated — rejected if malformed.
LLM WITH JSON SCHEMA
📚 Evidence Agent
Embeds query, retrieves from PubMed corpus. Drops everything below cosine 0.6. If nothing passes — returns insufficient_evidence with ZERO LLM calls.
RAG + THRESHOLD GATE
⚠️ Uncertainty Agent
Only activates for MEDIUM tier. Produces structured escalation report. Explicitly forbidden from making any clinical recommendation.
MEDIUM TIER ONLY
🛡️ Guard Agent
5 hard Python rules. Block/pass is boolean from pure logic — no LLM in the decision path. LLM may only explain why something was blocked, never decide it.
DETERMINISTIC ONLY
📄 Report Agent
Loads template with named slots. LLM copies validated content into slots only. System prompt explicitly forbids new clinical claims. No free narrative generation.

Guard Agent — The 5 Hard Rules

def run(result, diag, evidence, report_text):
    blocks = []
    reasons = []

    # Rule 1: Confidence above minimum gate
    if result.confidence < config.GUARD_MIN_CONFIDENCE:
        blocks.append("Confidence below guard threshold")

    # Rule 2: Every cited DOI must resolve (HEAD request to doi.org)
    for quote in evidence.get("quotes", []):
        doi = quote.get("doi")
        if doi and not _doi_resolves(doi):
            blocks.append(f"DOI {doi} does not resolve")

    # Rule 3: Clinical disclaimer must be present
    if config.CLINICAL_DISCLAIMER_SNIPPET not in report_text:
        blocks.append("Clinical disclaimer missing")

    # Rule 4: Diagnosis agent agrees with CNN
    if diag.get("agrees") is False:
        blocks.append("Diagnosis agent contradicts CNN output")

    # Rule 5: Evidence is valid or properly marked insufficient
    if evidence.get("status") not in ["ok", "insufficient_evidence"]:
        blocks.append("Evidence agent returned invalid status")

    guard_passed = len(blocks) == 0
    return GuardResult(passed=guard_passed, blocks=blocks, reasons=reasons)

RAG — Retrieval Augmented Generation

What Is RAG and Why Does It Matter Here?

RAG grounds the LLM's outputs in a verified document corpus. Instead of asking the LLM "what does the literature say about melanoma?" — which it might hallucinate — we embed a curated PubMed corpus into a vector database, retrieve the most relevant chunks for each case, and only let the LLM quote from what was retrieved. It cannot invent citations.

The Full RAG Pipeline

# ── INGESTION (one-time, offline) ──
1. Query NCBI PubMed E-utilities with dermoscopy MeSH terms
2. Fetch abstracts → 1,759 chunks (500 tokens, 50-token overlap)
3. Each chunk carries: {doi, pub_date, journal, source_type}
4. Embed with S-PubMedBERT-MS-MARCO → 768-dim vectors
5. Store in ChromaDB (persistent, local)

# ── RETRIEVAL (per inference) ──
query = f"{cnn_class} in {age}y {sex} on {localization}"
# e.g. "melanoma in 45y male on back"

query_embedding = embedder.encode(query)
results = chroma.query(query_embedding, n_results=5)

# Filter: drop everything below cosine similarity 0.60
valid_chunks = [r for r in results if r.score >= 0.6]

if not valid_chunks:
    return {"status": "insufficient_evidence"}
    # ← LLM is NEVER called. No hallucination possible.

# Only if chunks pass the threshold:
return llm.extract_quotes_only(valid_chunks)
# LLM prompt explicitly says: quote only, no synthesis, no new claims

Why S-PubMedBERT-MS-MARCO?

Standard BERT was trained on Wikipedia + BookCorpus — general English. S-PubMedBERT was fine-tuned on PubMed abstracts (biomedical domain) and then further fine-tuned on MS-MARCO (a passage retrieval task). This means it understands biomedical terminology like "melanocytic", "dermatoscopy", "keratinocyte" as domain-specific terms, not just vaguely medical-sounding words. The semantic similarity scores it produces are far more meaningful for our corpus than a general-purpose embedder.

If He Asks "Why Not 99%?"

Dermatologists themselves achieve approximately 77% on 7-class dermoscopy without clinical metadata or dermoscope access. We're at 73.9% using images only, with no patient history, no physical examination, no clinical context. The binary task at 89.6% is more clinically relevant anyway — the urgent question is always benign or malignant first.

Understanding the Errors (Confusion Matrix)

The largest source of confusion is nv (mole) classified as df (dermatofibroma) — approximately 294 cases. Both are benign. In binary terms, this is not an error at all — both map to the same benign class. Our binary accuracy is higher than our 7-class accuracy precisely because many 7-class confusions happen within the same binary group.

✅ The clinically dangerous error is a malignant case classified as benign. Our malignant sensitivity is 80.4% — we catch 4 out of 5 malignant lesions. Benign specificity is 95% — we only raise a false alarm on 1 in 20 truly benign cases.

Split Strategy — Why Lesion-ID Matters for HAM10000

HAM10000 has multiple images per lesion (same skin spot photographed multiple times). If you split by image, the same lesion can appear in both training AND test — the model memorizes the lesion and reports inflated accuracy. We split by lesion_id so every lesion exists in exactly one split. This is called leakage prevention. ISIC 2019 doesn't have patient IDs, so we did stratified random split by class.

Professor Q&A — Every Question He Will Ask

Click to Expand
⚡ Click each question to reveal the answer. Read these until you can answer without looking.
Q Why ViT instead of a standard CNN like ResNet or EfficientNet? ▼
Answer: A CNN's receptive field grows gradually through layers — early layers only see small local regions. ViT uses self-attention from the very first layer: every 16×16 patch attends to every other patch globally. For dermoscopy, the diagnostic signal is often the relationship between distant regions — for example, border irregularity relative to center pigmentation in melanoma. Self-attention models that directly. We also had 25,331 images which is enough to fine-tune a ViT without it overfitting.
Q Why is the confidence router rule-based and not LLM-based? ▼
Answer: Because routing is a safety-critical decision that must be deterministic, testable, and auditable. You can write a unit test that proves 0.4999 → LOW and 0.5000 → MEDIUM. You cannot write a unit test for an LLM's judgment — it's non-deterministic. In a clinical setting, if a case is misrouted and a patient is harmed, you need to explain exactly why. "The LLM decided" is not an acceptable answer to a medical ethics board.
Q What is temperature scaling and why is raw softmax not allowed? ▼
Answer: Neural networks trained with cross-entropy loss are systematically overconfident — they output high probabilities even on incorrect predictions. Temperature scaling divides the raw logits by a scalar T before softmax. T is fitted on the validation set to minimize negative log-likelihood. Our T=0.583 calibrated the model so that when it says 90% confident, it's actually right approximately 90% of the time. Raw softmax is forbidden from reaching the router because an overconfident raw score would incorrectly route uncertain cases into the HIGH tier.
Q What is ECE and what does 0.030 mean? ▼
Answer: Expected Calibration Error measures how well confidence scores match actual accuracy. You bin predictions by confidence range, compute the gap between average confidence and actual accuracy in each bin, and take the weighted average. ECE = 0.030 means our model's confidence is on average 3 percentage points off from its true accuracy. The target was below 0.05 — we achieved 0.030, which is well within that bound.
Q What stops the system from hallucinating medical citations? ▼
Answer: Three layers. First, the Evidence Agent can only query our offline ChromaDB corpus — it has no internet access. Second, if no chunk from the corpus scores cosine similarity ≥ 0.6 against the query, the function returns "insufficient_evidence" immediately and the LLM is never called at all. Third, if chunks do pass the threshold, the LLM receives a system prompt that explicitly forbids synthesis or new claims — it may only extract quotes that exist verbatim in the retrieved text. Finally, the Guard Agent resolves every DOI against doi.org before the report passes.
Q How did you handle class imbalance? ▼
Answer: Two mechanisms. WeightedRandomSampler oversamples minority classes during training so each epoch sees a more balanced distribution. Class-weighted CrossEntropyLoss penalizes mistakes on minority classes more heavily. We also capped the sampler at 10× the majority class frequency — without the cap, df (dermatofibroma, 115 training images) would be upsampled 72× and the model would memorize those 115 images rather than generalize.
Q Why 70 / 15 / 15 split? ▼
Answer: 70% training gives the model enough data to learn. 15% validation is used only for early stopping and temperature calibration — it never affects model weights directly. 15% test is held out completely until final evaluation. The test set was never seen in any form during training or calibration — it is a true held-out evaluation.
Q What is Grad-CAM / what are the attention maps? ▼
Answer: For CNNs, Grad-CAM computes gradient-weighted class activation maps — it backpropagates the gradient of the target class to the last convolutional feature map, producing a heatmap showing which spatial regions influenced the prediction most. For ViT, we use attention rollout: we aggregate the self-attention weights across all 12 transformer layers to produce a single attention map showing which patches the model "focused on." Both serve the same purpose — making the model's reasoning interpretable to a clinician.
Q Would this replace a dermatologist? ▼
Answer: Absolutely not. This is a clinical decision-support tool, not a diagnostic tool. Every output carries a mandatory clinical disclaimer. The system is designed to support the clinician's workflow — flagging high-confidence cases for attention, escalating uncertain cases for human review, and providing literature evidence. The final diagnostic decision always belongs to a licensed physician. Retraining the model itself is human-gated — no automatic updates can happen without a data scientist manually approving the new version.
Q Why not just use GPT-4 for everything? ▼
Answer: We deliberately constrained where LLMs operate based on what decisions need to be auditable. Routing and safety decisions are pure Python because they must be unit-testable and reproducible. Evidence retrieval is RAG-grounded because LLMs hallucinate citations. Report generation is template-bound because free narrative generation would introduce uncontrolled clinical claims. The LLMs we do use (Claude Haiku, Llama 3.3) only operate in well-scoped tasks: cross-referencing metadata, extracting existing quotes, and filling template slots. Every LLM call goes through a single anthropic_client.py wrapper for centralized auditing.
Q What would you improve if you had more time? ▼
Answer: Three things. First, ensemble the ViT with the VGG16 collaborator module — when two architectures trained on different data disagree on the binary label, that disagreement is itself a clinical signal worth escalating. Second, proper patient-ID grouped splitting for ISIC 2019 (some patients have multiple images from different angles — splitting by image slightly inflates test scores). Third, expand the RAG corpus beyond abstracts to full-text papers, which would improve evidence quality for rare classes like vasc and df.
Q Walk me through what happens when I upload an image. ▼
Answer: The image is loaded and preprocessed — resized to 224×224, normalized with ImageNet mean and standard deviation. It's passed through the fine-tuned ViT-Base-16 to produce 7-class logits. Those logits are divided by the temperature scalar T=0.583, then passed through softmax to get calibrated probabilities. The maximum probability across binary groups (benign/malignant) is the confidence score. The rule-based confidence router assigns a tier. For HIGH confidence cases, the Diagnosis Agent cross-references the prediction with patient metadata, the Evidence Agent retrieves relevant PubMed literature, the Guard Agent runs 5 deterministic safety checks, and the Report Agent fills a template with all validated outputs. The user sees the report along with the attention heatmap.
